Geopos Backlun

Main page - Backlun/front/geopos/index.html.

Static folder - Backlun/front/geopos/static/.

You can get the file from static folder at the address /src.

Example: Backlun/front/geopos/static/js/build/main.js -> /src/js/build/main.js

Test it: Backlun/back/geopos/_test_/request.test.js

Command required: npm install && npm run test

Geopos API

Params marked with asterisk (*) are required. The contents of the field "Response" is located in the field "body" in Server response object.

Example of response: {body: { ... }, message: 'Success', status: 0}

Method Address Params Response Description
GET /api/points/get - geostate* Geostate object Get all points in geostate
GET /api/points/get_from_token Token* string point* Point object Get a point of the token
POST /api/points/post point* Point object
 point* Point object
 Add a point to the geostate
GET /api/random_point/get - point* Point object Get random point
POST /api/random_point/post point* Point object
 point* Point object
 Add a random point to the geostate
GET /api/check_point/get - point* Point object Get the check point
PUT /api/check_point/put point* Point object
 point* Point object
 Put your point in geostate and return distance of checkPoint
POST /api/check_point/post point* Point object
 point* Point object Set new check point as primary goal

Standard objects

Server response:

status int
message string
body any

Status code: 0 - success, 1-9 - various errors, 10 - not found.

Point object:

Type: any
Token: string
Coordinats: [float64, float64]
example: { Type: 'Point', Token: 'Erzhn', Coordinates: [ 1.0511093343504192, 0.7650437151547923 ] }

Geostate:

Token: string
Point: Point object
example: {
VLdyv: { Type: 'Point', Token: 'VLdyv', Coordinates: [ 0.41411997330718403, 0.867472187530921 ] },
W2Vzz: { Type: 'Point', Token: 'W2Vzz', Coordinates: [ 1.3718466578497521, 1.3112190915139195 ] },
hJfmw: { Type: 'Point', Token: 'hJfmw', Coordinates: [ 0.9386107042591662, 1.0641967512107868 ] }
